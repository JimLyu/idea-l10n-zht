報告對 equals()、compareTo() 或比較一個物件是否與自身相等的類似方法的呼叫。 這些方法的方法約定指定了此類別呼叫將始終為 equals() 返回 true 或為 compareTo() 返回 0。 該檢查還會檢查對 Objects.equals()、Objects.deepEquals()、Arrays.equals()、Comparator.compare()、測試框架（JUnit、TestNG、AssertJ）的 assertEquals() 方法、Integer.compare()、Integer.compareUnsigned() 以及類似方法的呼叫。

請注意，在極少數情況下，該檢查可能會報告返回 false 的 equals() 呼叫，因為雖然兩邊的表達式相同，但它們會生成單獨的物件，並且會在參照上而不是在內容上執行比較。 最簡單的範例是 new Object().equals(new Object())。 無論如何，此類別呼叫都很可疑，而且可能具有其他意圖。

範例：


  class Foo {
    boolean foo(Object o) {
        return o.equals(o); // 警告
    }

    boolean bar(String[] ss) {
        return Arrays.equals(ss, ss); // 警告
    }
}
