報告用作 SLF4J 和 Log4j 2 日誌記錄方法的實參的字串模板。 方法 org.apache.logging.log4j.Logger.log() 僅支援所有日誌級別選項。 即使在沒有記錄日誌訊息的情況下，也會在執行時評估字串模板；這可能會對性能產生負面影響。 建議改用參數化日誌訊息，停用日誌記錄時不會對其進行評估。

範例（針對 Kotlin）：


      val variable1 = getVariable()
      logger.info("variable1: $variable1")

在套用了快速修復之後（針對 Kotlin）：


      val variable1 = getVariable()
      logger.info("variable1: {}", variable1)

Note that the suggested replacement might not be equivalent to the original code, for example, when string templates contain method calls or assignment expressions.

  • 使用警告物件列表可以忽略某些更高的日誌記錄級別。 可以始終啟用更高的日誌記錄級別，並且將始終評估實參。
  • Use the Do not warn when only expressions with primitive types, their wrappers or String are included option to ignore string templates, which contain only expressions with primitive types, their wrappers or String. 例如，它可能有助於防止載入延遲集合。 請注意，即使只使用具有基元類型的表達式建立字串，它們的包裝器或運行時的字串也會對性能產生負面影響。
