報告 suspend 函式內部的 runBlocking 呼叫。

在 suspend 函式中使用 runBlocking 會否定掛起機制，阻塞呼叫執行緒，並破壞異步編程的核心目的。

該快速修復會根據上下文，自動將 runBlocking 取代為以下選項之一：

  • run 呼叫。
  • 提供了特定 CoroutineContext 的情況下的 withContext 呼叫。
  • 內聯包裝程式碼

範例：


suspend fun something() {
  runBlocking {
    code() // 執行緒在此處被阻塞
  }
}

在套用快速修復後：


suspend fun something() {
  runBlocking {
    code() // 異步執行
  }
}
