Reports zip operations that may return Mono with void value inside a Reactive Stream chain.

Reactive Streams zip transformation methods with Mono Void parameters will not have any effect. The quick-fix suggests replacing zip(), zipWhen(), zipWith() with when, which returns Mono with Void value.

範例：

Mono.zip(Mono.just(1), Mono.fromRunnable(() -> System.out.println("value")))
                .map(v -> "New value")
                .doFinally(System.out::println);

在套用快速修復後：

Mono.when(Mono.just(1), Mono.fromRunnable(() -> System.out.println("value")))
                .then(Mono.fromCallable(() -> "New value"))
                .doFinally(System.out::println);
