報告重複的提供程序。

問題範例：


provider "kubernetes" {
  config_path = "~/.kube/config"
}

provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
}

解決此問題的其中一種方式是新增別名，並在所需的提供程序中建立資源時顯式指定它。

更正範例：


  provider "kubernetes" {
  config_path = "~/.kube/config"
}

provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
  alias = "cloud"
}
技術支援：Terraform 和 HCL 延伸模組
