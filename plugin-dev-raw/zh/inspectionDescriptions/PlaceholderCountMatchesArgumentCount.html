報告 SLF4J 或 Log4j 2 日誌記錄呼叫，例如 logger.info("{}: {}", key)，其中記錄器訊息中 {} 佔位符的數量與日誌呼叫的其他實參的數量不符合。

使用檢查選項指定 SLF4J 使用哪個實作。 如果選擇自動檢查，則在類別路徑中搜尋 org.apache.logging.slf4j.Log4jLogger。 如果找到了此檔案或選擇了是，則 SLFJ4 API 不會報告具有異常類型的最後一個參數具有佔位符的情況。 範例： 


// 不會報告
log.error("For id {}: {}", id, new RuntimeException());
此選項僅適用於 SLF4J。
