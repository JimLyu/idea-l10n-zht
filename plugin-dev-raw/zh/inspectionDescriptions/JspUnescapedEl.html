報告 JSP 頁面中未轉義的 EL 表達式中的非安全資料。 此類別情況可能導致跨站腳本攻擊 (XSS) 漏洞。

下面的描述使用術語 untainted 資料來指代安全可信的資料，與 tainted（不安全、不可信）相對。 請參閱 taint 檢查獲取更多資訊。

安全的 (untainted) 物件為：

  • 基元資料類型（char 除外）
  • 呼叫標記為 untainted 的方法的結果
  • 轉義函式呼叫的結果
  • 標記為 untainted 的欄位
對於所提供的資料，JSP 頁面可能有不同的上下文，如果資料對於一個上下文是安全的，並不一定意味著它對所有上下文都是安全的。 有關範例，請參閱跨站腳本攻擊防御備忘單。 此檢查會跳過在所有上下文中標記為 untainted（安全）的資料。

此檢查支援以下選項以將方法和欄位標記為安全：

  • 通過使用可組態的 untainted 註解列表中的任何註解
  • 通過組態安全方法和欄位列表

此類別檢查不會醒目提示 JSP 頁面中非安全資料到達安全標記的位置。 例如，JSTL taglib 中的 out 標記是安全的，因為它在預設情況下對給定值進行轉義。 安全標記列表是可組態的（省略 tag 或 attribute 值可定義整個範圍）。

"用 JSTL <out> 標記包裝" 快速修復會用 JSTL <out> 標記包裝 EL 表達式。

之前:

  
    ${foo}

之後:

  
  <%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
<c:out value="${foo}"/>

"用 JSTL escapeXml() 函式包裝" 會使用 escapeXml 轉義函式包裝非安全值。

之前:

  
  <%@ taglib prefix="custom" uri="WEB_INF/custom.tld" %>
  <custom:tag value="${foo}"/>

之後:

  
    <%@ taglib prefix="custom" uri="WEB_INF/custom.tld" %>
    <%@ taglib prefix="fn" uri="http://java.sun.com/jsp/jstl/functions" %>
    <custom:tag value="${fn:escapeXml(foo)}"/>

還有更多快速修復可以將所選標記新增到安全標記中，或者用 untainted 註解來註解相關方法（或欄位）。

可以新增自訂轉義函式（請參閱“轉移函式”頁籤）。 現在，它包含來自 JSTL 的返回安全值的函式（例如，已知的 escapeXml 函式）。 它不需要用不同的命名空間 URI 定義兩次 JSTL 函式。 如果使用了 "http://java.sun.com/jsp/jstl/functions" 或 "http://java.sun.com/jstl/functions" 中的一個，另一個也將被檢查。 也可以根據名稱在任何地方跳過轉義函式。 為此，請將命名空間 URI 欄位留空。
