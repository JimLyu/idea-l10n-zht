報告違反迪米特法則的情況。

迪米特法則並非真正的法則，但它指定了一種樣式準則：絕不呼叫從另一個呼叫中獲取的物件的方法。 The code that follows this guideline is easier to maintain, adapt, and refactor, has less coupling between methods, less duplication, and better information hiding. 另一方面，您可能需要編寫許多包裝器方法才能滿足這一准則。

範例：


  boolean pay(Customer c, Invoice invoice) {
    int dollars = c.getWallet().contents; // violation
    if (dollars >= invoice.getAmount()) {
      Wallet w = c.getWallet();
      w.subtract(invoice.getAmount()); // violation
      return true;
    }
    return false;
  }
上面的範例可以更好地實作為 Customer 中的 payInvoice(Invoice invoice) 方法。

Use the Ignore calls to library method and access of library fields option to ignore Law of Demeter violations that can't be fixed without changing a library.
