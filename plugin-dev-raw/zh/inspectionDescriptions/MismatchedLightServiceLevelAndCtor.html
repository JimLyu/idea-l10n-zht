報告輕量服務級別及其建構函式之間的不符合。

報告的問題如下：

  • 具有帶兩個類型為 com.intellij.openapi.project.Project 和 kotlinx.coroutines.CoroutineScope 的參數或者一個類型為 Project 的參數的建構函式的輕量服務類別不會被註解為專案級服務。
  • 註解為應用程式級服務的輕量服務類別沒有無實參建構函式，也沒有帶一個 CoroutineScope 類型參數的建構函式。

範例 (Kotlin)：


@Service                              // 建議在 '@Service' 註解中指定 'Service.Level.PROJECT' 參數
class MyService(val project: Project) // 建議從建構函式 'MyService' 中移除參數

套用建議在 '@Service' 註解中指定 'Service.Level.PROJECT' 參數的快速修復後：


@Service(Service.Level.PROJECT)
class MyService(val project: Project)

套用建議從建構函式 'MyService' 移除參數的快速修復後：


@Service
class MyService()
