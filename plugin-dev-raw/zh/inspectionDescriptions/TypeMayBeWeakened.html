報告可以更改為更抽象（更弱）類型的變數和方法返回類型。 這使程式碼更抽象，因此可重用性更高。

範例：


  // 參數類型可以削弱為 java.util.List
  void processList(ArrayList<String> list) {
    if (list.isEmpty()) return;
    System.out.println("Processing");
    for (String s : list) {
      System.out.println("String: " + s);
    }
  }

啟用下面的僅削弱為接口複選框，可僅在該類型可以削弱為接口類型時才報告問題。

啟用下面的不建議削弱宣告為 'var' 的變數複選框，可防止報告使用 'var' 關鍵字宣告的區域變數 (Java 10+)

停止類別意在防止類別削弱至低於停止類別的程度，即便可能出現這種情況，也能予以防止。 在某些情況下，這可能會提高可讀性。
