報告一個類別中的 SLF4J 或 Log4j2 日誌記錄呼叫，例如具有類似日誌訊息的 logger.info("message: {}", key)。 這些呼叫之間可能無法區分，這給理解某個日誌訊息來自何處帶來了困難。

範例 (Java)：


  private static void request1(String text) {
      log.info("Message: {}", text); //類似呼叫
      doSomething1();
  }

  private static void request2(int i) {
      log.info("Message: {}", i); //類似呼叫
      doSomething2();
  }
  • 使用相似序列的最小長度選項可設定相似序列的最小長度，之後將報告呼叫
  • 使用不報告日誌級別為 'error' 的呼叫選項可忽略日誌級別為 `error` 且存在異常的訊息。 隱藏警告可能會有用，因為仍然可以使用堆疊追蹤來定位呼叫站點
