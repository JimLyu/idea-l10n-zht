報告賦值後未使用的賦值。 如果不使用，最好移除該賦值，以縮短程式碼並避免冗餘分配。

報告的情況如下：

  • 該變數在賦值後從未被讀取。
  • 該變數在被讀取前始終被新值覆寫。
  • 變數初始設定式冗餘（出於以上兩種原因之一）。

組態檢查：

使用報告冗餘初始設定式選項可以報告冗餘的初始設定式： 


  int getI() {
    int i = 0; // 冗餘初始化
    i = 2;
    return i;
  }

使用在可以取代為(i + 1)時報告 ++i 選項，報告 ++i 表達式可以取代為 i + 1 的情況： 


  int preInc(int i) {
    int res = i;
    return ++res;
  }

使用在更改後的值後續未使用時報告 i++ 選項可以報告後續不使用 i++ 表達式的結果的情況： 


  int postInc(int i) {
    int res = i;
    return res++;
  }

使用報告其值從未被使用的模式變數選項來報告模式變數的值未被使用的情況： 


  if (object instanceof String s) {
      s = "hello";
      System.out.println(s);
  }

使用報告其值從未被使用的迭代參數選項來報告增強的 `for` 語句迭代參數的值未被使用的情況： 


  for (String  arg : args) {
      arg = "test";
      System.out.println(arg);
  }
