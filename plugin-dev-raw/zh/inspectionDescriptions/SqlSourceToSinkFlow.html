報告使用 Java 和 Kotlin 語言將非安全字串作為 SQL 查詢傳遞給方法的情況。 這可能是 SQL 注入的一個原因。 方法列表取自設定 - SQL、JPA QL、Hibernate QL 和 PostgreSQL 的語言注入

安全物件為：

  • 字串文字、接口實例或枚舉物件、int 及其包裝器、布爾及其包裝器、類別物件
  • 呼叫方法的結果，該方法的接收器和實參是安全的
  • 同一檔案中的 private 欄位，僅使用字串文字賦值，並具有安全的初始設定式
  • 同一檔案中的 final 欄位，具有安全的初始設定式
  • 從安全物件賦值的區域變數
    • 此欄位、區域變數或參數不得作為實參傳遞給方法或用作限定符，或者必須是基元、其包裝器或不可變。

分析僅在一個檔案內執行。 範例： 


    public void save(String sql) {
        JdbcTemplate jdbcTemplate = new JdbcTemplate();
        jdbcTemplate.queryForList(sql);
    }

  • 使用如果情況過於複雜導致無法檢查則報告選項來指定是否需要報告因為復雜度而無法檢查的字串

  • 使用將 private 方法的參數視為安全選項指定 private 方法的參數，否則它們將被視為未知。

  • 使用 Untainted 註解表來指定註解，這些註解將在分析過程中將被用作 @Untainted 註解。 @Untainted 宣告一個安全物件。
