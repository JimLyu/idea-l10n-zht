報告在以下情況下使用的 toString() 呼叫：
  • 在字串串聯中
  • 在 java.lang.StringBuilder#append() 或 java.lang.StringBuffer#append() 方法中
  • 在 java.io.PrintWriter 或 java.io.PrintStream 的方法中
  • 在 org.slf4j.Logger 的方法中

In these cases, conversion to string will be handled by the underlying library methods, and the explicit call to toString() is not needed. Removing redundant toString() calls can occasionally even improve performance and reduce object allocations.

範例：


  System.out.println(this.toString())

在套用快速修復後：


  System.out.println(this)

請注意，沒有 toString() 呼叫，程式碼語意可能會有所不同：如果表達式為 null，則將使用 null 字串，而不是拋出 NullPointerException。

使用 僅在已知限定符為非 null 時報告 選項以避免對可能為 null 的值發出警告。 Removing the explicit toString() in these cases will change the runtime semantics from throwing a NullPointException to silently accepting the value when it is null.
