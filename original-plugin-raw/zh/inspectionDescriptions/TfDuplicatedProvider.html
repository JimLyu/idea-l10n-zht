报告重复的提供程序。

问题示例：


provider "kubernetes" {
  config_path = "~/.kube/config"
}

provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
}

解决此问题的其中一种方式是添加别名，并在所需的提供程序中创建资源时显式指定它。

更正示例：


  provider "kubernetes" {
  config_path = "~/.kube/config"
}

provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
  alias = "cloud"
}
技术支持：Terraform 和 HCL 插件
