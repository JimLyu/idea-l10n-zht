报告用作 SLF4J 和 Log4j 2 日志记录方法的实参的字符串模板。 方法 org.apache.logging.log4j.Logger.log() 仅支持所有日志级别选项。 即使在没有记录日志消息的情况下，也会在运行时评估字符串模板；这可能会对性能产生负面影响。 建议改用形参化日志消息，禁用日志记录时不会对其进行评估。

示例（针对 Kotlin）：


      val variable1 = getVariable()
      logger.info("variable1: $variable1")

在应用了快速修复之后（针对 Kotlin）：


      val variable1 = getVariable()
      logger.info("variable1: {}", variable1)

Note that the suggested replacement might not be equivalent to the original code, for example, when string templates contain method calls or assignment expressions.

  • 使用警告对象列表可以忽略某些更高的日志记录级别。 可以始终启用更高的日志记录级别，并且将始终评估实参。
  • Use the Do not warn when only expressions with primitive types, their wrappers or String are included option to ignore string templates, which contain only expressions with primitive types, their wrappers or String. 例如，它可能有助于防止加载延迟集合。 请注意，即使只使用具有基元类型的表达式创建字符串，它们的包装器或运行时的字符串也会对性能产生负面影响。
